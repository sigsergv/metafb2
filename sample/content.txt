= Заголовок первого уровня
== Заголовок второго уровня (вложенная секция)

Уровень заголовка обозначается количеством символов «=» в начале строки.

Разделителями абзацев являются пустые строки. То есть данный
текст при компиляции будет преобразован
в один абзац.

Ставить пустую строку после строки с элементом-заголовком необязательно.

Каждый элемент-заголовок начинает новую секцию. Секция может содержать либо текст,
либо вложенные подсекции.

= Заголовок
= из двух строк

Заголовок может состоять из нескольких строк. Идущие подряд элементы заголовка с одинаковым
уровнем вложенности будут преобразованы в отдельные строки одного заголовка раздела.

=

Если после начальных символов «=» не идёт никакой текст, то данный раздел не будет иметь заголовка.

= Раздел с идентификатором
@id:identifier

Разделу может быть присвоен идентификатор, для этого сразу после строки-заголовка раздела
нужно поставить строку, начинающуюся со строки «@id:» и текстового идентификатора.

= Элементы оформления
== Оформление текста

Внутри текста можно выделять отдельные фрагменты как emphasis (используя //такое// обозначение)
или как strong (используя **такое** обозначение).

== Оформление эпиграфов (epigraph)
@e
Элемент //эпиграф// представляет собой несколько строк текста, ограниченные строками «@e» в начале
и «@e/какой-нибудь-текст». Текст «какой-нибудь-текст» будет использован в качестве имени автора для
эпиграфа.

Эпиграф может состоять из нескольких абзацев. Абзацы традиционно отделяются друг от друга пустой строкой.
@e/имя-автора

===
Эпиграф может стоять перед вложенным подразделом.

== Оформление аннотации (annotation)

@ann
Аннотация — это блок текста, который может стоять в начале раздела. Аннотация может стоять перед вложенным
подразделом.

Начало аннотации помечается строкой «@ann», окончание — строкой «@ann/».

Внутри блока аннотации могут встречаться блоки цитаты (cite), подзаголовка (subtitle), пустой строки (empty-line)
@ann/

== Оформление стихотворения (poem)

Блок стихотворения начинается со строки «@poem» и заканчивается строкой «@poem/имя-автора».

@poem
внутри стихотворения
идущие подряд строки
образуют строфу

пустая строка
означает границу
строфы

имя автора
опционально
@poem/автор

== Оформление цитаты (cite)

Цитата — это простой текстовый блок, который может содержать абзацы, подзаголовки (subtitle), а также
пустые строки (empty-line).

@cite
Блок цитаты начинается со строки «@cite» и заканчивается строкой «@cite/имя-автора»

Как обычно «имя автора» опционально
@cite/

== Оформление подзаголовка (subtitle)

Подзаголовок — это строка текста, начинающаяся с «@s:»

@s:вот это — подзаголовок

Подзаголовок может встречаться внутри блока и не разбивает текст на логические секции.

== Оформление пустой строки (empty-line)

Пустая строка — это строка, состоящая из «@empty-line». 

@empty-line

В FictionBook2 транслируется в элемент <empty-line/>

== Оформление иллюстраций

Иллюстрация добавляется в текст командой «@img:filename». filename здесь — это имя файла иллюстрации
в каталоге с картинками (каталог этот указывается в файле проекта).

@img:3op.jpg

== Оформление сносок/примечаний

Все примечания сноски содержатся в отдельном файле, имя этого файла указывается в файле проекта и обычно
это нечто вроде «notes.txt». По структуре это обычный файл в данной разметке, однако состоящий только из разделов
первого уровня, каждый раздел должен быть помечен уникальным идентификатором{{note-1}}. 

Чтобы вставить в документ ссылку на сноску/примечание, нужно добавить имя идентификатора сноски, заключённое 
в пару двойных фигурных скобок. В процессе трансляции все сноски будут перенумерованы и добавлены в конец итогового
документа.
